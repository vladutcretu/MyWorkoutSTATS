REST API - Documentation


If you need guidance to use the My Workout STATS - REST API, this is the right place.


Public endpoints

  • SignUp
    • URL: /api/signup/
    • Methods: POST
    • Informations and examples:
      • Example of valid data: {"username": "example", "email": "example@mail.com", "password": "examplepass"}
      • Successful request: Status -> 201 Created | Body -> { "message": "User account (via API endpoint) successfully created", "username": "example", "email": "examplepass" }
      • Failed request: Status -> 400 Bad Request | Body -> { Errors }

  • LogIn
    • URL: /api/login/
    • Methods: POST
    • Informations and examples:
      • Example of valid data: {"username": "example","password": "examplepass"}
      • Successful request: Status -> 200 OK | Body -> { "message": "Successfully logged in (via API endpoint).", "username": "example", "token": "2000531643fb..." }
      • Failed request: Status -> 400 Bad Request | Body -> { Errors }

  • LogOut
    • URL: /api/logout/
    • Methods: POST
    • Informations and examples:
      • Example of valid data: {"username": "example","password": "examplepass"}
      • Successful request: Status -> 200 OK | Body -> {"message": "Successfully logged out (via API endpoint)."}
      • Failed request: Status -> 403 Forbidden | Body -> { Errors }

Private endpoints

  • Profile
    • URL: /api/profile/
    • Methods: GET, PUT, PATCH, DELETE
    • Informations and examples:
      • Example of valid data: { "email": "example@email.com", "first_name": Test1, "last_name": Test2, "age": 18, "city": null, "bio": null, "instagram_url": "www.instagram.com" }
      • Successful request: Status -> 200 OK | Body -> {"message": "Profile updated successfully (via API endpoint)."}
      • Failed request: Status -> 400 Bad Request | Body -> { Errors }

  • MuscleGroups
    • URL: /api/musclegroups/ & /api/musclegroups/id/
    • Methods: GET, POST, PUT, PATCH, DELETE
    • Informations and examples:
      • Example of valid data: {"name": "Muscle group Example"}
      • Successful request: Status -> 201 Created | Body -> { Muscle group }
      • Failed request: Status -> 400 Bad Request | Body -> { Errors }

  • Exercises
    • URL: /api/exercises/ & /api/exercises/id/
    • Methods: GET, POST, PUT, PATCH, DELETE
    • Informations and examples:
      • Example of valid data: {"name": "Test Exercises", "musclegroup": "Existing musclegroup"}
      • Successful request: Status -> 201 Created | Body -> { Exercise }
      • Failed request: Status -> 400 Bad Request | Body -> { Errors }

  • Workouts
    • URL: /api/workouts/ & /api/workouts/id/
    • Methods: GET, POST, PUT, PATCH, DELETE
    • Informations and examples:
      • Example of valid data: {"name": "Workout example", "public": "yes" (OR "no"), "created": "2024-12-26"}
      • Successful request: Status -> 201 Created | Body -> { Workout }
      • Failed request: Status -> 400 Bad Request | Body -> { Errors }
